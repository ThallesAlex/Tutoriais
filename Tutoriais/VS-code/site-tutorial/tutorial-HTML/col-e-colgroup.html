Col e Colgroup

O que são?

<Colgroup>

  • <colgroup> define um grupo de colunas dentro de uma tabela e serve para aplicar formatação a uma ou colunas de forma centralizada

<col>

  • Col representa uma coluna individual dentro de um <colgroup> ou sozinho para aplicar atributos ou estilos a essa coluna específica

Onde colocar

  • O <colgroup> deve ser filho direto de <table>, após <caption>(se houver) e antes de <thead>, <tbody>, <tfoot> e qualquer <tr>

Atributos Importantes

  • span (em <col> ou <colgroup>) determina quantas colunas consecutivas o elemento cobre.
    Exemplo: <col span="2"> aplica a mesma configuração a duas colunas consecutivas
  • Você pode aplicar classes, estilos inline e atributos globais(como if, class) em <col> e <colgroup> para estilizar colunas inteiras

Por que usar

  • Evita repetir estilos célula a célula; útil para larguras, cores de fundo, alinhamento e bordas por coluna. Aplicar estilos via é mais limpo e semântico quando o objetivo é afetar colunas inteiras
  • Pode melhorar manutenção e clareza do HTML quando tabelas têm muitas células

Exemplo Prático

HTML

<table border="1">

    <colgroup>

        <col style="background-color:lightblue; width:150px;">

        <col style="background-color:lightgreen; width:200px;">

    </colgroup>

    <tr>

        <th>Nome</th>

        <th>Idade</th>

    </tr>

    <tr>

        <td>Ana</td>

        <td>30</td>

    </tr>

    <tr>

        <td>Bruno</td>

        <td>25</td>

    </tr>

</table>

Neste exemplo, a primeira coluna tem um fundo azul claro e largura de 150px, enquanto a segunda coluna tem um fundo verde claro e largura de 200px. Isso é conseguido usando os elementos <colgroup> e <col> para definir estilos específicos para cada coluna.(aplique ao seu código para ver, eu não quero ter qur fazer o exemplo mostrando o resultado)

Boas práticas e compatibilidade

  • Embora <col> e <colgroup> sejam amplamente suportados, sempre teste em diferentes navegadores para garantir a consistência visual
  • Use esses elementos para melhorar a semântica e manutenção do código, especialmente em tabelas complexas
  • Use <colgroup> e <col> quando a formatação é por coluna; para estilos complexos por célula continue usando classes em <td> ou <th> quando necessário
  • Lembre que nem todo estilo (por exemplo, conteúdo gerado com ::before em algumas células) é afetado por ; estilos aplicáveis via CSS a colunas são mais limitados que os aplicáveis a elementos de célula

Resumo Rápido

  • <colgroup> agrupa colunas para aplicar estilos ou atributos comuns
  • <col> define colunas individuais dentro de um <colgroup> ou sozinho
  • Use para estilizar colunas inteiras de forma eficiente e semântica
  • Melhora a manutenção do código em tabelas complexas
