Como Estilizar Listas Usando CSS

A aparência padrão das listas é simples, mas o CSS permite uma ampla gama de personalizações, tanto para o estilo dos marcadores quanto para disposição, cores, imagens e interações.

Propriedades principais para listas

  • list-style-type: Define o tipo dee marcador de uma lista
  • list-style-image: Usa uma imagem como marcador.
  • list-style-position: Controla onde o marcador é posicionado em relação ao texto.
  • list-style: Shorthand para combinar as três propriedades acima

Exemplos de uso:

Altera o tipo de marcador de uma lista não ordenada

CSS


ul {
list-style-type: square; /* opções: disc, circle, square
}

Lista ordenada com numeração romana

CSS


ol {
list-style-type: upper-roman; /* opções: decimal, lower-alpha, upper-alpha, lower-roman, etc. */
}

Usando umagem personalizada como marcador

CSS


ul {
list-style-image: url('icone.png');
}
  1. item 1
  2. item 2
  3. item 3

Dica: Sempre defina também um list-style-type como fallback para caso da imagem não ser carregada

Posição dos marcadores

CSS

ul{
list-style-position:inside;/* ou outside(padrão)*/
}
  • outside : marcador fica à esquerda do bloco do item.
  • inside : marcador fica dentro do bloco, alinhando em uma única linha ao texto.
Propriedade list-style (shorthand)

Você pode definir o tipo, posição e imagem em uma única linha:

CSS

ul {
   list-style: square inside url('icone.png');
}
Remover marcadores

CSS


ul {
   list-style: none;
   margim: 0;
   padding: 0;
}

Assim, todos os marcadores desaparecem e cada item pode ser estilizado individualmente

Customização Avançada: ::marker e pseudo-elementos

Com CSS moderno, é possível personalizar a cor, o tamanho, e até o conteúdo dos marcadores de lista com pseudo-elemento ::marker:

CSS


li::marker {
   color: hotpink;
   font-size: 1.5em;
   content: "→"; /* Adiciona seta como marcador */
}
  • item 1
  • item 2
  • item 3

Ou, para listas mais criativas:

CSS


li::marker {
 content: url (icone.svg);
}
O pseudo-elemento ::marker é compatível com a maioria dos navegadores modernos e permite estilização direcionada apenas ao marcador, sem afetar o restante do item

Acessibllidade, Semântica e Boas Práticas em Listas HTML

A estrutura semântica correta - usando <ul>, <ol>, <li>, <dl>, <dt> e <dd> conforme o propósito - garante que leitores de tela e outras tecnologias assistivas interpretem listas de modo ideal. Isso proporciona:

  • Leitura sequencial e lógica: leitores de tela anunciam quando uma lista começa, a quantidade de itens e o conteúdo de cada item.
  • Navegação eficiente: usuários podem saltar para lista rapidamente usando comandos do leitor de tela
  • Compatibilidade universal: mecanismos dee busca entendem a hierarquia do conteúdo e melhoram o SEO do site

Exemplos de boas práticas

  • Use <ul> para grupos de links, menus ou agrupamentos genéricos
  • Use <ol> para etapas, processos, índices ou listas em que a ordem/importância importa
  • Use <dl> para glossários, FAQs, propriedades de objeto e metadados
  • Evite criar listas usando apenas <div> e <span>, pois não fornecem significado ao conteúdo
  • Inclua textos claros e evite termos genéricos como "Clique aqui" em menus ou listas para melhor experiência de uso

Boas práticas para menus

Menus de navegação devem ser listados em <ul>, com cada link em um <li>. Para indicar um menu à tecnologia assitiva, coloque-o dentro de uma tag nav. Se remover os marcadores usando CSS adicione o atributo role= "list" para garantir o reconhecimento semântico em alguns navegadores:

HTML

<nav>
   <ul role="list">
      <li><a herf="destino do link">Home<a/></li>
      <li><a herf="destino do link">Serviços<a/></li>
   </ul>
</nav>
Em navegadores como Safari, listas estilizadas sem marcadores podem não ser reconhecidos pelo leitor de tela sem o atributo role

Listas de descrição e acessibildade

Em <dl>, o uso consiente de <dt> e <dd> matém a relaão semântica clara, ajudando leitores de tela a apresentar termos e suas definições de forma adequada.

Evite utilizar <dl> apenas para recuo visual ou formatação. Use-a sempre para representar pares de termo/descrição.

ARIA e listas HTML

A WAI-ARIA formece atributos e papéis que podem complementar a semântica padrão do HTML, por exemplo:

  • role="list": força o reconhecimento da lista como tal
  • aria-label, aria-labelledby, aria-describedby: adicionam acessíveis para listas complexas
  • Considerar ARIA apenaa quando não for possível usar semântica HTML adequada

Dicas de acessibilidade práticas

  • Menus e listas de navegação: sempre use <nav> para envolver listas de links principais.
  • Formulários: rótulos claros nos campos e listas auxiliares para instruções (ex: <ul> com instruções de preenchimento).
  • Textos alternativos: se usar imagens como marcadores, cetifique-se de não remover a informação de contexto do marcador para leitores de tela.
  • Contraste e legibilidade: ao estilizar listas, mantenha um alto contraste e legibilidade dos itens, especialmente para usuários com deficiência visual.

Boas Práticas de Semântica e Organização

Importância do HTML semântico

HTML semântico melhora:

  • Experiência do usuário: páginas bem estruturadas são mais fáceis de navegar e entender, especialmente com leitores de tela.
  • SEO mecanismos de busca priorizam páginas com estrutura semântica clara, melhorando o ranqueamento e a visibilidade do site
  • Manutenção: facilita atualizações e entendimento pro outros desenvolvedores

Exemplos de má prática

  • Usar apenas <div> e <span> para criar listas: prejudica a acessibilidade e a compreensão semântica
  • Colocar listas de títulos ou informações sem usar as tags apropriadas: dificulta navegação e indexação

Exemplo correto de uso semântico

HTML

<main>
   <h2>Passos para cadastro</h2>
   <ol>
      <li>Preencha o formulário</li>
      <li>Confirme o e-mail</li>
      <li>Acesse sua conta</conta>
   </ol>
</main>

Exemplo de uma FAQ com lista de definição

HTML

<section aria-labelledby="faq">
   <h2> id="faq">Perguntas Frequentes</h2>
   <dl>
      <dt>Como faço login?</dt>
      <dd>Utilize seu e-mail e senha cadastrados.</dd>
      <dt>Esqueci minha senha!</dt>
      <dd>Clique em "Esqueci minha senha" na página de login.</dd>
   </dl>
</section>
p{O uso adequado de etiquetas como <ol>, <ul>, <section>, <aria-labelledby> e <main> }
